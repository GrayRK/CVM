# ClaudeVoiceMaster (CVM) — Tasks

## Стадия 1 — Popup + Live State Inspector

**Цель**: рабочий интерфейс с мгновенным откликом и инструментом визуального тестирования.

### Popup (`entrypoints/popup/`)

Элементы управления:
- Поле ввода API ключа (сохраняется в chrome.storage.local)
- Выбор языка перевода (русский по умолчанию)
- Выбор голоса TTS (список фильтруется по выбранному языку)
- Toggle: озвучка вкл/выкл
- Toggle: субтитры вкл/выкл
- Слайдер: громкость TTS (0–100%)
- Слайдер: приглушение оригинала (0–80%)
- Индикатор статуса ("Готов" / "Переводим..." / "Ошибка")
- Кнопка открыть Developer Tools (</>)

**Требование**: все изменения применяются немедленно.
Никакой перезагрузки страницы.

### Интерфейс на странице с видео.

- Кнопка "Перевести и озвучить" (Запускает процесс)
- Субтитры внизу видео, белый текст на полупрозрачном фоне. Скруглённые углы.

### Live State Inspector (`entrypoints/devtools/`)

Отдельная страница расширения. Открывается кнопкой из popup.
Показывает все переменные состояния в реальном времени:

```
● videoVolume          = 0.4        Number    chrome.storage
● translationVolume    = 0.9        Number    chrome.storage
● subtitlesEnabled     = true       Boolean   chrome.storage
● ttsEnabled           = true       Boolean   chrome.storage
● selectedVoice        = "Dmitry"   String    chrome.storage
● targetLanguage       = "ru"       String    chrome.storage
● currentVideoId       = "dQw4w..."  String    runtime state
● translationStatus    = "ready"    String    runtime state
```

Обновляется через (долгосрочное соединение).
Каждое изменённое значение подсвечивается на 1 секунду.

### Ключевые моменты выполнения

**Каркас (отступления от исходной структуры, согласованы с пользователем):**
- `entrypoints/devtools/` → **`entrypoints/inspector/`**: имя `devtools` в WXT
  зарезервировано под настоящую панель Chrome DevTools (`devtools_page`),
  а нам нужна отдельная страница расширения, открываемая из popup.
- `content_bridge.ts` → **`content-bridge.content.ts`**: WXT распознаёт
  контент-скрипты только по шаблону `*.content.ts`.
- Добавлена папка **`lib/`** (`constants`, `types`, `storage`, `messaging`) —
  общий фундамент без магических констант и `any`.
- `wxt.config.ts`: `imports: false` — явные импорты из `#imports`, без авто-импортов.
- Везде унифицированный `browser` из `#imports` вместо глобального `chrome`.

**Popup:**
- Все настройки — в `chrome.storage.local` через типизированные
  `storage.defineItem` (`lib/storage.ts`); применяются мгновенно.
- Языки — короткий список (`ru, en, es, de, fr, zh, ja`).
- Голоса TTS читаются из `speechSynthesis.getVoices()` и фильтруются по языку;
  при смене языка список пересобирается, невалидный голос сбрасывается.
- API-ключ сохраняется с debounce; слайдеры хранятся как 0..1, отображаются в %.
- Добавлены (по запросу) два флага, по умолчанию выключены:
  `useYoutubeTranslation` (брать готовый перевод YouTube вместо API) и
  `autoStart` (запуск перевода при открытии). Логика — Стадия 2.

**Виджет на странице видео:**
- Кнопка вкл/выкл перевода в **Shadow DOM** (изоляция от стилей YouTube),
  монтируется поверх плеера (верх справа), уходит в фуллскрин с видео.
- Устойчивость к YouTube-SPA: монтаж по `yt-navigate-finish` + поллинг,
  таймеры/слушатели на `ctx` WXT (автоочистка при HMR).
- При SPA-навигации (другое видео / «назад») состояние сбрасывается в `false`.
- Контейнер субтитров **отложен до Стадии 2** (внешний вид настраивается на
  реальном тексте; пересекается со Стадией 1.1).

**Live State Inspector:**
- Отдельная страница `inspector.html`, открывается кнопкой `</>` из popup.
- Storage-значения обновляются через `storage.watch`; рантайм-значения
  (`translationActive`, `currentVideoId`, `translationStatus`) — через
  long-lived port из `background.ts`. Изменения подсвечиваются на 1 сек.
- API-ключ показывается замаскированным (не светим секрет).

**Известное ограничение (см. заметку в Стадии 2):**
- Рантайм-состояние в `background.ts` сейчас глобальное (одно на всё
  расширение). Для нескольких вкладок с видео нужно хранение per-tab —
  обязательная доработка Стадии 2.

## Стадия 2 — Текст из видео + расширение devtools

**Цель**: Получение, кеширование и отображение в devtools текста из видео.

Шаги:
- Получаем текст с YouTube видео на оригинальном языке
- Получаем текст автоперевода YouTube видео, которое делоает сам ютуб (По сути Google перевод)
- Реализуем кеширование обоих текстов. Сразу делаем логику (Не выполнять операции по вытаскиванию
текста с видео, если то существует в кэше)
- В devtools добавляем дополнительное меню:
  ->- Раскрывающийся список с позициями хранящий название, ссылку на видео, время создания.
  ->- Каждая позиция так-же раскрывается и хранит три банера с текстом. Для оригинального текста,
   автоперевода, и полученного перевода с API (Логику API банера будем реализовывать в стадии 3, пока заготовка)
  ->- В раскрывающийся список добавляем сортировку по времени создания.
  ->- При раскрывании определённой позиции сворачиваем старую развёрнутую (Что-бы не загромождать пространство)
  ->- Добавить в данный раздел кнопку позволяющую очистить все кэшированые тексты.

**Примечание**  Старт работы завязан на кнопку  вкл/выкл перевода (реализованую на Стадии 1), а не при старте страницы.

### Ключевые моменты выполнения

**Источник текста — ПЕРЕХВАТ запроса плеера (а не прямой fetch):**
- Сначала пробовали прямой `fetch` по `captionTracks[].baseUrl` из
  `movie_player.getPlayerResponse()` — YouTube отдавал **200 + 0 байт**:
  у этих URL нет `pot` (proof-of-origin token), который веб-клиент обязан
  прикладывать. Подделать его нельзя.
- Итог: мост (`content-bridge`, MAIN) патчит `window.fetch` и
  `XMLHttpRequest` на `document_start` и перехватывает **реальный запрос
  плеера** к `/api/timedtext` (там валидный `pot`) — берёт оттуда готовое
  тело оригинала. Перехват один на страницу (guard-флаг на `window`).
- Автоперевод YouTube (Google): к перехваченному URL добавляем
  `&tlang=<язык>&fmt=json3` — `pot` тот же, запрос отрабатывает.
- Парсинг с автоопределением формата: `json3` (тайминги `tStartMs`/
  `dDurationMs` — задел под озвучку Стадии 4) с фолбэком на «сырой» XML.

**Авто-включение субтитров (чтобы было что перехватывать):**
- По запросу из content мост: если CC выключены — включает их (API плеера
  `setOption('captions',…)`, иначе **клик по родной кнопке** `.ytp-subtitles-button`),
  ждёт перехвата (поллинг до 6 с), затем **возвращает как было** (выключает,
  если включал он). На время операции прячет контейнер субтитров CSS-ом —
  на экране ничего не мелькает.

**Каналы связи:**
- Мост(MAIN) ↔ content(ISOLATED): `window.postMessage`, конверт
  `{ source: 'cvm-bridge', message }`; одноразовый `request-extraction` →
  `extraction-data` (videoId, title, язык, перехваченные URL+тело) с таймаутом.
- content → background: `runtime.sendMessage` с ответом — `cache-lookup`
  (есть ли запись / есть ли перевод на язык) и `cache-store` (запись).
- Inspector ↔ background: long-lived порт; `runtime-state` / `cache-list` /
  `cache-entry` (вниз) и команды `request-cache-list` / `request-cache-entry`
  / `clear-cache` (вверх).

**Модель кэша (`lib/cache.ts`) — ОДНА запись на видео:**
- Ключ `cvm_cap_{videoId}` (без языка) + лёгкий индекс `cvm_cap_index`.
  Внутри записи: оригинал один раз, словарь автопереводов
  `translations: { ru: [...], es: [...] }`, заготовка `apiTranslation: null`.
- Префикс `cvm_cap_` отделён от зарезервированного под Стадию 3 ключа API
  `cvm_v_{videoId}_{lang}`.
- Смена целевого языка **не плодит запись**, а добавляет язык
  (`upsertEntry`). Если язык уже есть — извлечение пропускается.
- Если целевой язык == оригиналу (по базовому коду, `en-US`≈`en`) —
  автоперевод не нужен.
- `readIndex` устойчив к повреждённым данным (отбрасывает невалидные
  элементы) — иначе битый индекс ронял запись и «Очистить кэш».

**Флаг «Использовать автоперевод» (`useYoutubeTranslation`):**
- Включён → кэшируем оригинал + автоперевод на целевой язык.
- Выключен → кэшируем **только оригинал** (+ заглушка API); автоперевод не
  тянется (оригинал нужен Стадии 3 для перевода через API).

**Авто-синхронизация при активном виджете:**
- Пока виджет включён, `storage.watch` на `targetLanguage` и
  `useYoutubeTranslation` сам до-кэширует новый язык — без передёргивания
  кнопки. Запуски пайплайна сериализованы (без наложения; перезапуск с
  актуальными настройками).

**Автозапуск (`autoStart`):**
- При открытии `/watch` и при SPA-навигации на новое видео виджет сам
  переходит в «включено» и запускает пайплайн (если флаг включён).

**Меню кэша в Inspector:**
- Аккордеон под таблицей: позиции (название, ссылка `↗`, время), раскрытие
  одной сворачивает предыдущую (видимость через класс `.open`, не атрибут
  `hidden`). Внутри — баннеры: оригинал + по одному на каждый язык +
  заглушка API.
- Кнопка-переключатель сортировки по времени создания (новые↔старые).
- Полный текст позиции подгружается при раскрытии (`request-cache-entry`),
  список обновляется живо. Кнопка «Очистить кэш» (с подтверждением).

**Окружение dev (`wxt.config.ts`):**
- Постоянный профиль Chrome (`~/.cvm-chrome-profile`, `keepProfileChanges`) —
  cookie-согласие/сессия YouTube сохраняются между запусками (иначе чистый
  профиль не отдаёт субтитры). Папка ВНЕ проекта — иначе Vite-вотчер падает
  на залоченных Chrome-файлах.
- Флаг `--disable-blink-features=AutomationControlled` — маскировка
  автоматизации, YouTube отдаёт обычную сессию.

**Статус**: ✅ Выполнена

### Ключевые моменты выполнения

**3.1 — Ядро перевода (`background.ts`):**
- Перевод запускается в ветке `useYoutubeTranslation = OFF` пайплайна
  (`content.ts`): ON → автоперевод YouTube (Стадия 2), OFF → Claude API.
- Запрос к `/v1/messages` (`claude-sonnet-4-6`), заголовки `x-api-key` +
  `anthropic-version` + `anthropic-dangerous-direct-browser-access`.
  Для fetch из фона добавлен `host_permissions: https://aiprimetech.io/*`.
- Батчинг (см. 3.4 про размер), пул на **3** воркера, `withRetry` — 1 повтор
  ТОЛЬКО на сетевые/HTTP-ошибки (не на формат).
- **Нумерованный протокол вместо строгого JSON.** Реплики уходят как
  `[1] текст`, модель возвращает `[n] перевод`. Сборка по номерам: потерянный
  номер → подставляется оригинал. Длина результата всегда = числу сегментов
  (тайминги 1:1 для Стадии 4), несовпадение количества НЕ рушит батч и не
  плодит retry-шторм. Решило баг «длина 128 ≠ 150» и «8 запросов на 1/4».
- Хранение per-language: `apiTranslations: Record<lang, CaptionSegment[]>`
  (по аналогии с `translations`), в мете `apiLanguages[]`.
- Индикатор `N/M` на кнопке виджета (фон шлёт прогресс в tab через
  `tabs.sendMessage`) + рантайм-строка `translationProgress` в Inspector.

**3.2 — API Monitor (Inspector):**
- Метрики собираются по ходу перевода и пишутся в запись кэша per-language:
  `apiMeta: Record<lang, ApiTranslationMeta>` (модель, число батчей, символы,
  токены из `usage`, тайминги per-batch и общий, `videoSeconds`). Переживают
  перезапуск SW.
- У баннера «Перевод Claude API · {язык}» — кнопка **📊 Монитор** → модалка с
  таблицей метрик (закрытие: крестик / фон / Esc).
- ⚠️ Точность метрик требует доработки (см. Стадия 3.4).

**3.3 — Калькулятор стоимости:**
- Эмпирическая калибровка вместо хардкода цен: калькулятор учит `R = $/символ`
  и `D = символов/мин` усреднением по выборкам ТЕКУЩЕЙ модели (база — символы
  оригинала). Цены за токены у пользователя НЕ запрашиваются.
- Фиксация в модалке монитора: поле `$` + «Зафиксировать» → валидация →
  `confirm` → блокировка поля. Стоимость пишется и в `apiMeta[lang].costUsd`,
  и в отдельное хранилище `cvm_cost_samples` (переживает «Очистить кэш»).
  Фиксацию выполняет фон (`record-api-cost`), сам берёт объём/токены/длительность
  из `apiMeta`. Повторная фиксация запрещена (`already-fixed`).
- Калькулятор на странице Inspector: три зависимых контрола (длина видео —
  ползунок; символы и стоимость — точный ввод), связаны через `R`/`D`; живут
  через `storage.watch` на выборках. Кнопка «Сбросить калибровку».
- Тема — тёмная, в стиле Inspector (образец взят как референс по структуре).
- ⚠️ При корректных входных данных результат неправдоподобен — баг (Стадия 3.4).

## Стадия 4 — TTS

**Цель**: озвучка переведённых субтитров синхронно с видео.

### Edge TTS (старт)

На Windows 11 Microsoft Neural голоса доступны через Web Speech API.
Проверка при запуске:

```typescript
const voices = speechSynthesis.getVoices()
const dmitry = voices.find(v => v.name.includes('Dmitry') && v.lang === 'ru-RU')
```

Если Neural голоса не найдены — fallback на любой ru-RU с уведомлением пользователю.

### Приглушение оригинала

Снижать громкость видео на значение из настроек пока говорит TTS.
Плавное восстановление (transition 300ms).
